# Mover el onboarding antes del login

**Features**

- [x] Mostrar el onboarding solo la primera vez que se abre la app después de instalarla.
- [x] Enseñar el onboarding antes de mostrar las opciones para entrar con Apple o Google.
- [x] Mantener un flujo de 3 pasos, corto y claro.
- [x] Ocultar la opción de saltarlo para que toda persona vea la introducción completa.
- [x] Incluir un paso con una animación donde varios stats entran solos y se acomodan dentro del canvas.
- [x] Al terminar el onboarding, mostrar la pantalla de acceso con Apple y Google.
- [x] Después de iniciar sesión, llevar a la experiencia normal donde aparecen las tarjetas de conexión.

**Design**

- [x] Estilo oscuro editorial, premium y elegante.
- [x] Pantallas limpias, con mucho aire, tipografía protagonista y textos breves.
- [x] Movimiento suave y cinematográfico, sin verse recargado.
- [x] Indicador de progreso discreto para que se sienta guiado.
- [x] Superficies con profundidad sutil para mantener el look nativo y refinado.

**Pages / Screens**

- [x] **Paso 1 — Introducción:** presenta la promesa principal de la app con una entrada visual fuerte.
- [x] **Paso 2 — Canvas en acción:** muestra los stats cayendo y acomodándose sobre el canvas para explicar el valor del editor.
- [x] **Paso 3 — Preparado para empezar:** cierra la historia y empuja naturalmente hacia el acceso con Apple o Google.
- [x] **Pantalla de acceso:** aparece justo después de completar el onboarding.
- [x] **Pantalla siguiente:** tras iniciar sesión, recién ahí se muestran las tarjetas de conexión de Strava, Garmin y COROS.

**Experience details**

- [x] La primera apertura debe sentirse curada y aspiracional antes de pedir acceso.
- [x] Las siguientes veces, el onboarding ya no aparece.
- [x] La transición del último paso hacia el acceso debe sentirse continua y premium.
- [x] La animación del canvas debe ser uno de los momentos más memorables de toda la entrada a la app.
